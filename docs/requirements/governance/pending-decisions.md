# 코드 감사 후 미결정 사항

> 상태: Closed — 현재 미결정 제품 정책 0건  
> 기준일: 2026-07-20  
> 목적: 코드와 요구사항을 대조하면서 제품 의도만으로 확정할 수 없었던 정책을 한 곳에 모은다.  
> 원칙: 이 문서의 항목은 요구사항 ID가 아니다. 결정 전에는 명명된 Policy/Port 뒤에 격리하고 데이터 손실이 없는 동작을 선택한다.

## 3. 낮은 우선순위의 UX·호환 정책

| 번호 | 결정할 질문 | 권장안 | 결정 전 안전 동작 | 영향 경계 |
|---|---|---|---|---|

Q-001은 DEC-063으로 확정했고 Q-002는 기존 DEC-011과 중복임을 확인해 최초 활성화 정책으로 통합했습니다. Q-003은 DEC-017·DEC-052에 일반 사용자 복구 금지와 운영 복구일 기준 자동화 재개 정책으로 확정했습니다. Q-004는 DEC-033에 원문 최초 응답 1회·동일 발급 요청의 `AlreadyIssued` 메타데이터 재생·명시적 재발급 정책으로 확정했습니다. Q-005는 현재 카드별 확인이 별도 Reporting 차원이 아니라 Ledger의 카드 문자열 검색과 검색 결과 월별 합계로 충족되고, 등록 카드에 사용자 자유 입력 이름도 없음을 코드에서 확인해 미결정 제품 기능에서 제거했습니다. 해당 계약은 `SEA-002`·`SEA-004`와 DEC-059 보완으로 명시했습니다. Q-006은 필수 gate 실패를 waiver로 우회하지 않는 정책으로 [DEC-064](decisions.md#dec-064)에 확정했습니다. 현재 남은 미결정 사항은 없습니다.

## 4. 결정 처리 규칙

1. 답변을 받으면 [결정 기록](decisions.md)에 새 DEC를 추가하거나 기존 DEC를 수정합니다.
2. 소유 모듈 `requirements.md`의 상태·예외와 `design.md`의 Policy·테스트를 같은 변경에서 갱신합니다.
3. 결정 전 테스트는 권장 결과를 정답으로 고정하지 않고 “데이터 손실·권한 우회·비결정적 결과가 없음”만 검증합니다.
4. 단순 구현 선택은 제품 질문으로 다시 올리지 않습니다. SDK, 저장소, Queue, Adapter는 이 문서의 제품 결과를 만족하는 범위에서 상세 설계가 결정합니다.
